#### 核心优势：匿名命名空间 > 全局 `static`

| 特性        | 全局 `static`       | 匿名命名空间               |
| ----------- | ------------------- | -------------------------- |
| 支持特性    | 仅变量 / 函数       | 支持变量、函数、类、模板等 |
| 语义清晰度  | 依赖 C 语言兼容规则 | 纯 C++ 语义，可读性更强    |
| 嵌套 / 组合 | 不支持              | 可嵌套命名空间，灵活度高   |





| 维度         | 具名命名空间（如 `namespace BSL {}`）   | 匿名命名空间（`namespace {}`）                               |
| ------------ | :-------------------------------------- | ------------------------------------------------------------ |
| 名称         | 有明确自定义名称（如 BSL、Net、String） | 无名称（编译器生成唯一内部名）                               |
| 作用域范围   | 全局作用域（整个程序可见）              | 仅当前编译单元（`.cpp` 文件）可见                            |
| 跨文件访问性 | 可通过 `extern`/`using` 跨文件访问      | 无法跨文件访问（`extern` 也穿透不了）                        |
| 核心用途     | 模块化组织代码、避免全局命名冲突        | 实现「文件私有」，替代全局 `static`                          |
| 支持元素     | 变量、函数、类、模板、嵌套命名空间等    | 同左（支持所有 C++ 元素）                                    |
| 链接属性     | 内部元素默认「外部链接」（可跨文件）    | 内部元素为「外部链接」但命名空间唯一不可见（效果等价内部链接） |
